# 제품 요구사항 정의서 (PRD)

## 1. 프로젝트 개요
- **프로젝트명**: Arithmetic Puzzle (셈셈퍼즐)
- **목표**: 남녀노소 누구나 즐길 수 있는 두뇌 회전 사칙연산 퍼즐 웹 게임 개발
- **플랫폼**: 모바일 웹 (세로 모드 전용)
- **디자인 컨셉**: 밝고 캐주얼하며 트렌디하고 세련된 UI/UX (첨부 이미지 참조)

## 2. 핵심 기능

### 2.1 게임 플레이 및 규칙
- **게임 방식**:
  - 가로/세로로 연결된 수식(Crossword 형태)의 빈칸을 채워 모든 수식을 완성해야 함.
  - 빈칸에 들어갈 숫자는 하단에 제공되는 **'숫자 카드 풀(Pool)'**에서 선택하여 배치함.
  - 제공된 모든 숫자 카드를 사용해야 레벨 클리어 가능.
  - 연산자(+, -, ×, ÷)와 등호(=)는 고정되어 있음.
  - **숫자 범위**: 모든 칸(피연산자 및 결과값 포함)에 들어가는 숫자는 **0 이상 100 이하**의 정수여야 함.

- **조작 방식 (Input Method)**:
  - **Drag & Drop**: 하단의 숫자 카드를 드래그하여 빈칸에 놓음.
  - **Tap & Place**: 빈칸을 먼저 선택하고 하단의 카드를 탭하여 놓음 (모바일 편의성).
  - **이동 및 교체**: 이미 놓여진 카드를 다시 드래그하여 다른 빈칸으로 옮기거나, 풀(Pool)로 되돌릴 수 있음.

- **피드백 (Validation)**:
  - 카드를 빈칸에 놓는 **즉시** 정답 여부를 판별함.
  - **정답 시**: 카드가 해당 칸에 고정되거나, 성공 시각 효과(파란색/초록색 등) 표시.
  - **오답 시**: 
    - 카드가 튕겨져 나가거나, 붉은색 효과와 함께 풀(Pool)로 돌아감.
    - 상단의 **하트(Life)**가 1개 감소함.

- **승리 조건**: 
  - 모든 빈칸을 올바른 숫자로 채우고, 모든 수식이 성립할 때.

- **패배 조건**:
  - 하트 3개 모두 소진 시 게임 오버. (광고 보고 이어하기 등의 기능 확장 가능성)

### 2.2 게임 아이템 (Power-ups)
하단에 2가지 아이템 제공 (이미지의 퍼즐 조각 수집 기능은 제외):

1.  **돋보기 (Hint)**: 
    - 사용 시, 플레이어가 **빈칸 하나를 선택**하면 해당 칸에 들어갈 정답 카드를 찾아 자동으로 채워줌.
2.  **전구 (Smart Fill)**:
    - 사용 시, 남은 빈칸 중 **가장 큰 숫자**가 들어갈 칸을 자동으로 찾아 채워줌.

### 2.3 레벨 및 데이터
- **JSON 기반 레벨 구성**: 
  - 각 레벨은 고정된 보드 형태와 정답 데이터를 가짐.
  - 레벨이 올라갈수록 빈칸 수 증가, 수식 구조 복잡화, 숫자 범위 확대 등으로 난이도 조절.
- **데이터 구조**:
  - `Level ID`, `Difficulty`, `Board Layout` (Grid), `Solution Map`, `Initial Fixed Cells`.

### 2.4 사용자 관리 (Supabase)
- **로그인/회원가입**: 소셜 로그인 (Google, Kakao 등).
- **데이터 저장**:
  - 현재 최고 도달 레벨.
  - 획득한 별(코인) 개수.
  - 보유 아이템 개수.
  - 각 레벨별 클리어 여부 (별점 등).

## 3. 정보 구조 (IA) 및 UI 상세

### 3.1 상단 헤더 (Header)
- **코인**: 현재 보유 재화 표시.
- **레벨**: 현재 레벨 번호 (예: 레벨 62).
- **생명력(하트)**: 남은 기회 표시 (3개).
- **설정 버튼**: 게임 일시정지 및 설정 메뉴 진입.

### 3.2 메인 게임 영역 (Board)
- **퍼즐 보드**: 
  - 격자(Grid) 형태의 수식 배열.
  - 숫자 칸(Input Cell), 고정 숫자 칸(Fixed Cell), 연산자 칸(Operator Cell), 빈 칸(Empty/Wall).
  - 정답 입력 됨/안됨 상태 시각적 구분.

### 3.3 하단 컨트롤 영역
- **숫자 카드 풀 (Number Pool)**: 
  - 사용 가능한 숫자 카드들이 나열됨.
  - 남은 카드 개수와 종류를 한눈에 볼 수 있어야 함.
- **아이템 버튼**: 
  - 돋보기, 전구 아이콘 및 남은 횟수 뱃지.

## 4. 데이터 구조 예시

```json
{
  "level": 62,
  "difficulty": "Hard",
  "boardSize": { "rows": 8, "cols": 7 },
  "grid": [
    // r: row, c: col, type: cell type, value: displayed value (if fixed/operator), answer: correct value (if input)
    { "r": 0, "c": 0, "type": "input", "answer": 12 },
    { "r": 0, "c": 1, "type": "operator", "value": "-" },
    { "r": 0, "c": 2, "type": "fixed", "value": 9 },
    { "r": 0, "c": 3, "type": "operator", "value": "=" },
    { "r": 0, "c": 4, "type": "input", "answer": 3 }
  ],
  "pool": [1, 1, 2, 3, 5, 12, 12, 15, 16] // 정답을 구성하는 모든 숫자들
}
```

## 5. 기술 스택
- **Framework**: Next.js 14+ (App Router)
- **Styling**: Tailwind CSS (Custom Design System)
- **State Management**: Zustand (Game validation logic, Drag & Drop state)
- **Backend/Auth**: Supabase
- **Animation**: Framer Motion
- **Icons**: Lucide React
