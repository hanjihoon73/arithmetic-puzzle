# 구현 계획 - 퍼즐 자동 생성기 (Puzzle Generator)

## 목표
13x20 크기의 그리드에 수학 수식 퍼즐을 자동으로 생성하는 `Generator` 모듈 구현.

## 기술 명세

### 1. 파일 구조
- `src/lib/generator.ts`: 핵심 생성 로직 및 타입 정의.
- `src/types/index.ts`: 필요한 경우 생성기 관련 타입 추가.

### 2. 핵심 알고리즘 (Crossword Style)
1. **그리드 초기화**: 13(cols) x 20(rows) 크기의 빈 2차원 배열 생성.
2. **첫 번째 수식 배치**:
   - 랜덤한 수식 (`A op B = C`) 생성.
   - 그리드 중앙 부근이나 랜덤 위치에 가로/세로 방향으로 배치.
3. **추가 수식 배치 (반복)**:
   - **타겟 선택**: 이미 배치된 수식의 숫자/셀 중 하나를 '교차점'으로 선택.
   - **수식 생성**: 교차점의 숫자를 포함하는 새로운 유효한 수식을 생성.
     - 예: 교차점 숫자가 `5`라면, `5 + ? = ?` 또는 `? - 5 = ?` 등의 형태.
   - **배치 시도**:
     - 교차점을 기준으로 가로/세로 방향 결정 (기존 수식과 수직 방향).
     - 그리드 경계를 벗어나지 않는지 확인.
     - 기존의 다른 수식과 인접 문제(충돌, 모호함)가 없는지 확인.
   - **성공 시**: 그리드에 기록하고 다음 단계로.
   - **실패 시**: 다른 교차점이나 다른 수식으로 재시도 (백트래킹/재시도 제한).

### 3. 난이도 조절 (Difficulty Scaling)
- `generateLevel(difficulty: 'easy' | 'medium' | 'hard')` 함수 제공.
- **Easy**: `+`, `-` 위주. 작은 숫자(1~10). 수식 길이 짧음.
- **Medium**: 곱셈 포함 가능. 중간 숫자(1~20).
- **Hard**: `*`, `/` 적극 사용. 큰 숫자. `A + B + C = D` 형태 시도.

### 4. 레벨 데이터 변환
- 생성된 그리드를 게임에서 사용하는 `Level` 객체로 변환.
- **빈 칸 처리**: 완성된 수식에서 일정 비율의 셀을 `type: 'input'`으로 변경하여 플레이어가 맞춰야 할 문제로 만듦.
  - 난이도에 따라 빈 칸 비율 조절 (Easy: 30%, Hard: 50% 등).
- **Number Pool 생성**: 정답이 필요한 숫자들을 모아서 `pool` 배열 생성.

## 구현 단계
1. `src/lib/generator.ts` 기본 뼈대 작성 (타입, 상수).
2. 수식 생성 유틸리티 (`generateEquation`) 구현.
3. 그리드 충돌 감지 및 배치 로직 (`placeEquation`) 구현.
4. 메인 생성 함수 (`generatePaddle`) 구현.
5. 게임 스토어와 연동하여 '새 게임' 시 자동 생성된 레벨 로드.
